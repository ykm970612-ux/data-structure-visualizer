시각화 자료구조 설명

자료구조 핵심 개념

이 페이지는 시각화 페이지에서 다루는 네 가지 자료구조의 개념, 주요 연산, 시간복잡도, 사용 예시를 정리한 학습용 설명 페이지입니다.

Stack

Stack은 마지막에 들어온 데이터가 가장 먼저 나가는 LIFO 구조입니다. 데이터는 top 위치에서만 삽입되고 삭제됩니다.

  • 주요 연산: push, pop, top
  • 시간복잡도: push O(1), pop O(1), top O(1)
  • 예시: 브라우저 뒤로 가기, 실행 취소, 함수 호출 스택

Queue

Queue는 먼저 들어온 데이터가 먼저 나가는 FIFO 구조입니다. 데이터는 rear에서 들어오고 front에서 나갑니다.

  • 주요 연산: enqueue, dequeue, front, rear
  • 시간복잡도: enqueue O(1), dequeue O(1)
  • 예시: 프린터 대기열, 작업 스케줄링, BFS 탐색

Linked List

Linked List는 데이터와 다음 노드의 주소를 함께 저장하는 노드들이 연결된 구조입니다. 배열처럼 연속된 메모리를 필요로 하지 않습니다.

  • 주요 연산: 노드 삽입, 노드 삭제, 노드 검색
  • 시간복잡도: 검색 O(n), 삭제 O(n), 삽입 위치에 따라 O(1) 또는 O(n)
  • 예시: 음악 재생 목록, 동적 메모리 관리, 해시 체이닝

Binary Search Tree

Binary Search Tree는 왼쪽 자식은 부모보다 작고 오른쪽 자식은 부모보다 큰 값을 가지는 이진 트리입니다. 중위 순회를 하면 값이 오름차순으로 나옵니다.

  • 주요 연산: insert, search, inorder traversal
  • 시간복잡도: 평균 O(log n), 한쪽으로 치우치면 최악 O(n)
  • 예시: 정렬된 데이터 탐색, 집합 표현, 검색 인덱스
